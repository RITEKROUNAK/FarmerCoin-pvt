Getting Started for React Strapi CMS

How to install strapi CMS in your local
  1. Node.js and NPM : You can download Node.js from https://nodejs.org. please use Recommended For Most Users 16.13.2 LTS NPM comes bundled with Node.js.
  2. Enter to gigaland-cms directory
  3. Run npm install
  4. After that, run npm run develop to deploy the cms project in your local directory
  5. Open http://localhost:1337/admin to enter the cms dashboard
  6. Change password strapi with new password in your profile setting
    default user: strapi.admin@mail.com , password : Strapisuperadmin1
  7. Enter directory gigalandlightgrey / gigaland-retro
  8. Run npm install
  9. Run npm install webpack@4.44.2
  10. After that, run npm start for work in local your directory at localhost:3000
  11. Run npm run build to build the project. The build artifacts will be stored in the build/ directory. Use build directory for upload in your server
Note: Please run npm run develop for gigaland-cms directory & run npm start for front directory gigaland / gigaland-grey / gigaland-retro in same time for work
If not work in front directory using NPM try using Yarn
  1. Enter directory gigalandlightgrey / gigaland-retro
  2. add SKIP_PREFLIGHT_CHECK=true in file .env
  3. Remove node module command rm -rf node_modules
  4. Delete package-lock.json
  5. Install Yarn ( https://yarnpkg.com/en/docs/install ).
  6. Run yarn install
  7. Run yarn add webpack@4.44.2
  8. Run yarn run start from the folder
Install Gigaland-nft Minting with Metamask
  1. to start using this functionality you have to register an account on alchemy website and pinata first, you will have to use the API key and Secret to start using it - https://www.alchemy.com/ - https://app.pinata.cloud/
  2. also install and create an account for metamask, you will have to configure it to use the ropsten test network
  3. go to https://faucet.metamask.io/ to get some test ETH to use
  4. after installing the metamask follow this article to get your private key https://metamask.zendesk.com/hc/en-us/articles/360015289632-How-to-Export-an-Account-Private-Key
  5. after that, you can open the gigaland-nft folder, this folder is needed to create the smart contract that we will deploy to the ethereum network
  6. find the .env file, you will have to copy your alchemy API url and metamask private key
  7. install all the dependencies with npm install, then run npx hardhat compile
  8. run npx hardhat run scripts/deploy.js --network ropsten
  9. you will see the contract address printed on the command line, keep this address so we can use it to mint our NFT
  10. open the gigaland folder, and find the .env file. we will copy the Pinata KEY and SECRET also the Alchemy API url into the file
  11. one last file need to be changed is the interact.js file inside src/core/nft/interact.js you will have to change this line `const contractAddress = "0x92d474d216ce429c1efc7f48d7ff66bae5d070cd";` with the address you keep before
  12. you can start developing and test the minting process by going to the minting page
  13. more references: - https://ethereum.org/en/developers/tutorials/how-to-mint-an-nft/
How to use Gigaland Redux (Dummy API using json files)

this version uses Dummy API (Fake Data) so expect some of the function exclusive to Strapi CMS will not function properly.
this function include the Protected Route, Register, Login and Profile page

you can find the Dummy API Response in the .json files inside public/mock_data/ folder. you can modify these files to change the data shown in the Template
remember to keep the json structure as is, if you modify the json structure then you might have to change the front end code to match your changes.

if you want to remove the Dummy API and use your own API for the front end then you can check on this file to change the API endpoint

How to use Gigaland Redux CMS (Strapi)

this version uses Strapi CMS API to provide response for the front end side. if you're not familiar with Strapi please check out their documentation.
note that we are currently using the latest strapi v3 version. so please refer to the Strapi v3 documentation
to use the CMS please refer to the step above on how to get the Strapi CMS working in you local environment.

on this version the Protected Route, Register, Login and Profile page is working by using Strapi built in function.

Protected Route is used to block unauthorized user from accessing a certain page defined in the routes. you can find Authentication related files in src/core/auth folder. 

            // file location: src/components/app.js
            ...
            const ProtectedRoute = ({ component: Component, ...rest }) => {
              const isAuth = auth.getToken() !== null;
              
              return (
                  isAuth ? <Component {...rest} /> : <Redirect from="" to="/login" noThrow />
              )
            };
            ...
            ...
            ...
            // With this route definition, the user must be logged in first before they can access Author and Profile Page
            <ProtectedRoute component={Author} path="/Author/:authorId"/>
            <ProtectedRoute component={Profile} path="/Profile/:authorId"/>
Register, Login and Profile is used to Register a new user for the CMS, this new user will have a default Author collection created upon registering.
once you register for a user, you can use it to Login into the CMS from the front end login page. or you can use this existing credential to Login 

            Email : gigaland_admin@mail.com
            Password : GigalandAdmin

            Email : profile_update@mail.com
            Password : TestProfileUpdate
after a successful Login then you can access the Profile page, this page is used to update the User Profile Info. you can upload a profile picture and a banner for the user and update the user information

Custom background image for div/section

To make custom background image without touching css file, you can use code style="url(your-image-url)" within your div/section tag. See sample below:

style={{backgroundImage: `url(${'./img/background/4.jpg'})`}}
